# MilesingLight

MilesingLight is a public continuation seed for **Milesing's Light**: one authored mathematical world, portrayed in two and three dimensions through a private-first native Windows application.

This repository is public on purpose. It preserves the work, its authorship, and buildable source so one machine, account, or host is never the sole point of survival. Lawful mirrors, forks, study, rebuilding, and continuation are welcome under the copyleft licenses. Keep the authorship, licenses, provenance, and hashes with the work.

> Current release: `v0.1.0-alpha`, an unsigned Windows x64 development preview. It is a substantial working foundation and an honest beginning, not a claim that the complete future system already exists.

## Where this goes

The direction is one complete authored formation that can be edited closely in 2D, encountered and navigated in 3D, derived through traceable Milesing operations, recalled at purpose-appropriate detail, and kept under the author's control without requiring a server or cloud account.

Each Light unit retains its authored identity while routing through a finite typed operation, spatially designated operands, and an inspectable expression receipt. The application is meant to grow from the Light rather than display the Light as detached decoration.

The reviewed design routes are published under [docs/design](docs/design). They describe the concept space learned from native Windows engineering, diagram interaction, spatial engines, storage, recovery, and the 111-operation workbench. The Draw.io and Godot reference trees themselves are absent.

## What works now

- One native C++20 Windows application built from seven source bodies.
- All 111 Light units appear once with branch, operator, signature, mathematical origin, and arity.
- Every entry has a bounded finite executor and a constructive domain fixture; 111/111 pass.
- One authored world appears across **MY LIGHT**, **2D FRACTAL**, **3D FRACTAL**, **OPERATIONS**, **TIME / MEDIA**, and **SYSTEM RELATION**.
- The 2D and 3D portrayals share authored node identities rather than making detached copies.
- Pan, zoom, orbit, attention, movement, framing, bounded portrayal, and authored-order transformations are live.
- The Operations workspace selects all Light operations, edits supported bounded operands, executes typed fixtures, and can feed a compatible prior result into another operation.
- Canonical `.mlight` files preserve the authored world, relations, applied operations, expression routes, replay-derived results, and current attention.
- Saving uses a flushed candidate, protected prior, replacement, SHA-256 verification, semantic reopen, and bounded recovery inventory.

See [STATUS.md](STATUS.md) for the exact implemented/future boundary and verification evidence.

## Privacy boundary

The current executable has no HTTP, socket, server, account, telemetry, updater, registry-writing, service, or child-process route. Its imports are limited to ordinary Windows windowing, graphics, text, COM, cryptography, and filesystem support.

The application writes through explicit Save or Save As actions. Personal `.mlight` workspaces, machine logs, private development instructions, internal work notes, and third-party reference trees are absent from this repository.

## Run the preview

Requirements:

- Windows 10 or Windows 11, x64
- Direct3D 11-capable graphics
- No installer, account, server, or network connection

Release asset:

```text
File: MilesingLight.exe
Size: 1,072,128 bytes
SHA-256: 376A92B080979A10228027412D51413994D664BF9E13367AC2227A7AFD44ED12
```

The executable is unsigned, so Windows reputation or antivirus systems may inspect or warn about it. Verify the checksum and, for strongest assurance, inspect and build the source.

Useful controls:

| Input | Action |
|---|---|
| `F1`-`F6` | Change workspace |
| Left-click / drag | Attend and move a Light body |
| `Ctrl` + click | Toggle attention |
| `Shift` + drag in 2D | Marquee attention |
| Middle/right drag | Pan in 2D; orbit in 3D |
| Mouse wheel | Zoom, dolly, or move through Operations |
| `F` | Frame the authored world |
| `Ctrl+O` | Open `.mlight` |
| `Ctrl+S` / `Ctrl+Shift+S` | Save / Save As |
| `Ctrl+Z` / `Ctrl+Y` | Undo / redo |
| Operations: arrows, `Tab`, `+/-`, `V`, `R`, `Enter` | Choose, adjust, feed, reset, execute |

## Build and verify

Use an x64 MSVC C++ developer environment with the Windows SDK. The `.txt` source extensions are intentional; `/TP` compiles `APP.txt` as C++.

Exact release and self-test commands are in [BUILDING.md](BUILDING.md). A successful self-test reports:

```text

PASS canonical codec: every Operand tag + 111 applied fixtures round-trip byte-exactly
PASS recoverable local writing: first + replacement + protected prior + SHA-256 + exact reopen
PASS canonical realization: attention + exact Editor restoration + replay-derived results
```
