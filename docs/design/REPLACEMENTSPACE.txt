REPLACEMENT CONCEPT SPACE

STATUS
The live version-2 path writes and verifies one complete canonical text body.
Chunk/resource manifests, concurrent revision freezing, advance space planning
and exact-artifact cleanup preview remain expansion work.

PURPOSE
A save commits one complete authored work while preserving a known complete prior body.

LIVE DIRECT SAVE FLOW
1. Form one canonical version-2 body from the current authored world, shared
   Editor runtime, applied receipts and attention.
2. Parse and replay that complete body before writing any bytes.
3. Create a uniquely named candidate beside the destination.
4. Write exact bytes and call FlushFileBuffers.
5. Reopen the candidate; compare exact bytes, SHA-256 and semantic replay.
6. If no destination exists, move the candidate to the final path.  Otherwise
   call ReplaceFileW with an explicit protected-prior path.
7. Reopen the final path; compare exact bytes, SHA-256 and semantic replay again.
8. Return the final/candidate/prior paths, byte extent, SHA-256 and reopen facts;
   then prune only verified old canonical priors within bounded inspection.

EXPANDED SAVE TARGET
Add deliberate free-space planning, a frozen revision while concurrent editing
continues, chunk/resource manifests, richer outcome inventory and visible
artifact cleanup decisions without weakening the live exact-body chain.
