REPLACEMENT CONCEPT SPACE

PURPOSE
A save commits one complete authored work while preserving a known complete prior body.

DIRECT SAVE FLOW
1. Freeze one authored revision reference while editing may continue into a later revision.
2. Calculate the exact candidate size and confirm room for current work, candidate, protected prior body, and evidence.
3. Create a uniquely named candidate beside the destination on the same local volume.
4. Write header, chunks, resources, and manifest with exact byte counts.
5. Flush the candidate handle.
6. Close and reopen the candidate through the bounded parser.
7. Verify hashes and compare the complete logical root.
8. If no current work exists, move the verified candidate to the final path and reopen it.
9. If a current work exists, call ReplaceFileW with an explicit protected-prior path.
10. Handle every documented ReplaceFileW outcome by inventorying the final, candidate, and prior paths before offering another action.
11. Reopen the final path, verify it, and retain the write receipt.
12. Release obsolete candidates only through an explicit exact-artifact preview.
