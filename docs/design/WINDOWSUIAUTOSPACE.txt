WINDOWS UI AUTOMATION CONCEPT SPACE

STATUS
This is the accessibility implementation target.  V0.2 already has keyboard
workspaces, focus portrayal, readable scaling, high-contrast/reduced-motion
preference hooks and semantic editor commands.  A native UI Automation provider
tree and the patterns below are not attributed to the release until their
provider source and executable accessibility checks exist.

PURPOSE
The assistive encounter reaches the same Light, authored worlds, commands, text, mathematics, portals, timeline, writing, and recovery as the visual encounter.

ORIGINAL PROVIDER BODY
- WM_GETOBJECT exposes one root IRawElementProviderSimple for the Milesing window.
- The provider tree is generated from the current semantic workspace and authored portrayal, not from invisible renderer objects.
- Each reachable command exposes a stable name, purpose, keyboard route, enabled result, and Invoke pattern.
- Workspaces, panels, portals, authored bodies, relations, operand sockets, timeline tracks, and evidence surfaces expose appropriate control types.
- Focus changes follow the same authored attention path and raise UIA focus events.
- Selection and multi-attention expose Selection/SelectionItem where the authored command supports them.
- Editable properties expose Value or RangeValue.
- Expandable authored regions expose ExpandCollapse.
- Scrollable worlds expose Scroll while retaining their authored camera/viewport meaning.
- Text and mathematical editors expose Text, TextEdit, Selection, caret, range movement, attributes, and offscreen document access.
- Generated status and operation results raise concise live-region announcements.
- Readable scale, system contrast, visible focus, and reduced motion alter portrayal while keeping authored meaning available.

CODE CONNECTION
- EDITOR.txt is associated with the semantic tree and command descriptions.
- SUPPORT.txt is associated with exact text, expressions, authored bodies, and relation paths.
- RENDERER.txt consumes accessibility emphasis and visual preferences.
- APP.txt answers WM_GETOBJECT and forwards focus/event changes.
