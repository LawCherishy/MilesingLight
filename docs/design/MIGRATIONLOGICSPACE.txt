LIVE VERSION-1 TO VERSION-2 FLOW

- Require each original NODE self-number / operator-code / origin tuple to match
  its exact published catalog route, including only the enumerated historical
  origin spellings.

  its current named operation; stored result and note text cannot dictate the
  current mathematical result.
- Retain original IDs, authored geometry, relations, order, expression routes,
  operand designations and explicit attention.  An early file without ATTENTION
  is admitted only when a valid SIZE receipt proves the selected self numbers.
- Associate the new LIGHT TREATMENT satellite with the existing REALITY body
  while leaving every original route in place.
- Validate the resulting 112-self-number workspace.  The source file remains
  unchanged; an explicit later save emits canonical version 2 through the normal
  protected-candidate route.
- Version-2 input must reproduce exact canonical version-2 text after replay.

EXPANDED MIGRATION TARGET

containing source/destination generation, transformations, authored counts,
exact Light routes, hashes, candidate/prior paths and reopen results.  Unknown
future records are currently rejected rather than silently retained.
